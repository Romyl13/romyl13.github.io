Мене звати Роман

Я студент 2 курсу ЗУНУ (Західноукраїнського національного університету). Захоплювався веб-розробкою ще з шкільних років. Маю практичний досвід у створенні сайтів протягом року, володію HTML, CSS, JavaScript, jQuery, Bootstrap 4/5, Sass/Scss/Less. Досвідчений користувач Git та GitHub. Спеціалізуюсь на фронтенді, постійно вдосконалююсь у сфері веб-технологій.

HTML5

HTML5 - це ніби скелет веб-сторінки: він визначає, які елементи (заголовки, параграфи, зображення тощо) будуть на сторінці і як вони будуть розташовані.

CSS3

CSS3 – це інструмент для створення красивих інтер'єрів для веб-сторінок. За допомогою нього можна вибирати кольори, шрифти, розташування елементів і багато іншого.

JavaScript

JavaScript – це мова програмування, яка додає інтерактивність та динаміку до статичних веб-сторінок. JavaScript надає веб-сторінці життя та функціональність.

Вміння

Що я використовую для роботи
  • HTML5
    Цей інструмент слугує основою для вашого веб-проекту. Завдяки новій версії ми зможемо оптимізувати структуру сайту під вимоги пошукових систем, що зробить його більш помітним для потенційних клієнтів.
  • CSS3
    За допомогою цієї мови стилів я можу реалізувати будь-який дизайн для вашого веб-сайту або додатку.
  • JavaScript
    Цей мова програмування дозволяє оживити будь-що: слайдери, модальні вікна, підказки, вкладки, отримувати дані з сервера та багато іншого.
  • jQuery
    Завдяки бібліотеці jQuery ми зможемо швидше створювати веб-сайти. Ми не будемо її використовувати без потреби, але знаємо, як з нею працювати.
  • React
    Ця бібліотека дозволяє розробляти веб-додатки. Ми можемо створити максимально інтерактивний продукт, спеціально підлаштований під ваші завдання.
  • Node JS
    "Ця платформа надає готові інструменти для створення бекенду вашого продукту – невидимої для користувача частини, яка відповідає за обробку даних і логіку додатку.
  • MongoDB
    Це спеціальна база даних, де будуть зберігатися всі дані вашого сайту або програми.
  • TypeScript
    TypeScript – допомагає визначити, який «матеріал» використовувати в кожній частині програми. Він допомагає визначити помилки і зробити розробку швидшою.
